# 🧪 React Fusion State v1.4.0 — Interactive Demos

This directory contains interactive demonstrations of React Fusion State features. **Zero dependencies, maximum performance with ultra-simple API.**

> All demos load `react-fusion-state.umd.js` (auto-regenerated by `npm run build` from the published `dist/`), so what you see is exactly what npm consumers ship. **No deprecation warnings should appear in the console** when running any of these demos — if one does, please open an issue.

## 📁 Available Demos

### **demo-persistence.html** — v1.0 API showcase
- **Complete feature showcase** - All core React Fusion State capabilities
- **Global and per-key persistence** examples
- **Multiple components** sharing state
- **Real-time state synchronization**
- **Console logging and debugging**
- **Error handling** and fallbacks

### **demo-selectors.html** — v1.2 Selectors API ⭐ NEW
- **`useFusionStore(selector, equalityFn?)`** in action — derived values across keys
- **`shallow`** equality function for multi-key object selectors
- **Visual render counters** prove that selector components only re-render when their *selected* slice changes — sibling mutations on unrelated keys leave them alone
- Real-world pattern: a cart total that only updates when items actually change

### **demo-multi-store.html** — v1.4 Multi-store ⭐ NEW
- Two completely **independent `createStore()`** instances side by side (one cart store, one theme store)
- **Headless action button** that calls `cartStore.setState({...})` from outside any React Provider — proves the store works without React
- **Cross-store isolation indicator** confirms that mutating one store never re-renders components wired to the other

## 🎯 What Each Demo Shows

### Key Persistence Features Demonstrated

#### **Granular Control**
```jsx
// Configure persistence at Provider level (RECOMMENDED)
<FusionStateProvider 
  persistence={['user', 'settings']}
  debug={true}
>
  <App />
</FusionStateProvider>

function App() {
  // Persistent state (configured at provider level)
  const [user, setUser] = useFusionState('user', {});
  
  // Also persistent (configured at provider level)
  const [settings, setSettings] = useFusionState('settings', {});
  
  // Not persistent (not in persistence array)
  const [temp, setTemp] = useFusionState('temp', '');
}
```

#### **Custom Options**
- **debounceTime**: Control save frequency (0ms to 2000ms)
- **debug**: Enable/disable console logging
- **adapter**: Custom storage backends (auto-detected)

## 🔧 Customization

### Adding New Components
```jsx
// Configure at provider level (RECOMMENDED)
<FusionStateProvider 
  persistence={['myData']} 
  debug={true}
>
  <MyComponent />
</FusionStateProvider>

function MyComponent() {
  const [data, setData] = useFusionState('myData', {});
  return <div>{/* Your component */}</div>;
}
```

### Changing Persistence Settings
```jsx
// Simple granular persistence (RECOMMENDED)
<FusionStateProvider persistence={['counter', 'name']}>
  <App />
</FusionStateProvider>

// Advanced configuration
<FusionStateProvider persistence={{
  persistKeys: ['counter', 'name'],
  debounce: 1000,
}}>
  <App />
</FusionStateProvider>

function App() {
  const [state, setState] = useFusionState('counter', 0);
  // State automatically persisted based on provider config
}
```

## 🐛 Troubleshooting

### Demo Not Working?
1. **Check browser console** for errors
2. **Ensure JavaScript is enabled**
3. **Try a local server** instead of file:// protocol
4. **Clear localStorage** if behavior seems stuck

### Persistence Not Working?
1. **Check localStorage support** in browser
2. **Verify debug mode** for console logs
3. **Try incognito/private mode**
4. **Check storage quotas** (rare)

### Key Persistence Specific Issues
1. **Debug logs not showing**: Check `debug: true` option
2. **Debounced saves not working**: Wait for debounce delay
3. **Wrong storage keys**: Check browser localStorage in DevTools
4. **Data not loading**: Check browser's localStorage in DevTools
